# Quick Task 260403-k85: Inline Subtask Creation with Focused Input

**One-liner:** Inline auto-focused input row for subtask creation with Enter/Escape/blur UX, hierarchy guard hiding '+' for Epic and Subtask types, todoStateId fallback to first project state, and error logging for all silent catch blocks.

## What Was Built

### Task 1: task-detail-panel.tsx

- **`onAddSubtask` prop signature changed** from `() => void` to `(title: string) => void` in `TaskDetailPanelProps`, `SectionSubtasks`, and propagated through component tree
- **`isAdding` state + `inputRef`** added to `SectionSubtasks` — clicking '+' sets `isAdding(true)` instead of immediately calling `onAddSubtask`
- **Auto-focus**: `useEffect` focuses the `<input>` when `isAdding` becomes `true`
- **Inline input row**: plain `<input>` with semantic tokens (`border-primary`, `bg-primary`, `text-primary`, `text-placeholder`, `border-brand` on focus)
  - `Enter` key: commits non-empty value via `onAddSubtask(title)`, clears `isAdding`
  - `Escape` key: cancels without API call
  - `onBlur`: commits non-empty, cancels on empty
- **`handleCommit` useCallback**: extracts commit logic to avoid stale closure issues
- **Hierarchy guard**: `canAddSubtask = taskType !== 'epic' && taskType !== 'subtask'` — '+' button only renders when `canAddSubtask && onAddSubtask`
- **`taskType` prop** passed to `SectionSubtasks`; section visibility condition updated to include hierarchy guard
- **Duplicate '+' removed** from `SectionTaskHeader` — the function and prop `onAddSubtask` removed from that component entirely; no more two-button confusion

### Task 2: Both page files (sales-pipeline + tasks)

- **`handlePanelAddSubtask(title: string)`**: now accepts the typed title instead of hardcoding `'New subtask'`; wrapped in `try/catch` with `console.error('[handlePanelAddSubtask] Failed...')`
- **`todoStateId` fallback**: `projectStates[0]?.state_id ?? null` when no `'to-do'` category state exists — prevents silent no-op when board uses non-standard category names
- **8 silent `.catch(() => {})` replaced** per page (16 total) with named `console.error` calls:
  - `[board-load]`, `[agent-load]`, `[project-states-load]`, `[add-card]`, `[signed-urls]`, `[add-comment]`, `[upload-attachment]`, `[delete-attachment]`
- **Recovery catches preserved**: `.catch(() => { refetch() })` patterns on column rename/reorder and patchCard left intact — they already provide meaningful recovery
